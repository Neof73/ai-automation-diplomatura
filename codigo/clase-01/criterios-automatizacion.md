# Framework de 4 criterios para evaluar si un proceso se automatiza

## Los 4 criterios

Para cada proceso, responder SÍ (1 punto) o NO (0 puntos):

### 1. ¿Es repetitivo?
El proceso se ejecuta de la misma forma cada vez, sin variaciones significativas.

**SÍ:** registrar un email en una planilla, enviar una confirmación de compra, copiar datos de un formulario.
**NO:** redactar una propuesta comercial personalizada, negociar con un proveedor, diseñar una estrategia.

---

### 2. ¿Usa datos estructurados?
El proceso trabaja con información que tiene un formato predecible: números, fechas, campos de formulario, estados de una lista.

**SÍ:** el número de pedido, la fecha de compra, el estado (pendiente / en camino / entregado), el email del cliente.
**NO:** una conversación compleja con un cliente difícil, el diseño de un producto, la interpretación de un contrato.

---

### 3. ¿Tiene reglas claras?
Las decisiones del proceso se pueden describir con reglas del tipo "si X entonces Y".

**SÍ:** "si el estado es pendiente, enviar alerta", "si el cliente es VIP, aplicar 10% de descuento".
**NO:** "decidir si el cliente tiene razón en su reclamo", "evaluar si un candidato es adecuado para el puesto".

---

### 4. ¿Tiene volumen alto?
El proceso ocurre con suficiente frecuencia como para que la automatización genere ahorro significativo.

**Regla práctica:** si el proceso se hace menos de 5 veces por semana y tarda menos de 10 minutos, el ROI puede ser bajo.

---

## Cómo puntuar

| Puntaje | Interpretación | Acción |
|---|---|---|
| 4/4 | Automatización ideal | Prioridad máxima |
| 3/4 | Muy buena candidata | Alta prioridad |
| 2/4 | Automatizable con condiciones | Evaluar el ROI |
| 1/4 | Difícil de automatizar completamente | Solo automatizar partes |
| 0/4 | No automatizable | No invertir tiempo en esto |

---
