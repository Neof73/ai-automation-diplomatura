# Clase 19 — Taller final: pulir, testear y preparar la demo

## Archivos de esta clase

| Archivo | Descripción |
|---|---|
| `testing/checklist-pre-demo.md` | Checklist técnica completa antes del Demo Day |
| `demo/mensaje-de-prueba.txt` | El mensaje de prueba principal para la demo en vivo |
| `demo/mensaje-respaldo.txt` | Mensaje de respaldo si el principal falla |
| `demo/guion-10-minutos.md` | El guion de presentación de 10 minutos |
| `docs/documentacion-sistema-template.md` | Template para documentar el sistema |

## Qué hacer hoy

**Primera hora:** ejecutar la checklist completa de `testing/checklist-pre-demo.md`

**Segunda hora:** ensayo de la demo con el grupo
1. Preparar los archivos de `demo/` en VS Code
2. Ensayo de 3 minutos en parejas
3. Dos demos completas frente al grupo con feedback

## La regla del Demo Day

Un sistema funcionando mediocremente supera a un sistema perfecto que falla en vivo.
El objetivo de hoy es garantizar que la demo funciona, no que sea perfecta.
